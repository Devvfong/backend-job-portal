# Job Portal Backend – Project Progress

## Project Overview
This project is a **Job Portal Backend System** that connects job seekers with companies.  
The system allows users to browse jobs, apply for positions, and allows companies to create and manage job postings.

The backend is designed using **Node.js, Express, Prisma ORM, and PostgreSQL (NeonDB)** following a clean layered architecture.

---

# Tech Stack

## Backend
- Node.js
- Express.js
- Prisma ORM
- PostgreSQL (NeonDB)

## Authentication
- JWT (JSON Web Tokens)

## Development Tools
- Git
- GitHub
- Nodemon

---

# Architecture Pattern

The backend follows a **layered architecture**:


Routes → Controllers → Services → Prisma → Database


### Routes
Handle API endpoints and map them to controllers.

### Controllers
Handle request and response logic.

### Services
Contain business logic and Prisma database queries.

### Prisma
ORM used to interact with the PostgreSQL database.

# Database Design

The system uses four main entities.

### Entities


User
Company
Job
Application


### Relationships


Company 1 → N Jobs
User 1 → N Applications
Job 1 → N Applications


### Constraint

A **unique constraint** prevents a user from applying to the same job multiple times.
